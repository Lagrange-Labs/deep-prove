LayerNorm

Description of the Layer

LayerNorm can be thought of as transforming a tensor so that its values follow a normalised standard distribution, where we normalise on a certain dimension. For example if we have an input tensor ‘A‘ defined over the reals ‘R‘. Without loss of generality we can assume ‘a∈Rk×n‘ is a matrix then we define

LayerNorm(A)i,j​:=γ⋅n1​⋅∑l=1n​(Ai,l​−μi​)2+ϵ​Ai,j​−μi​​+β.​

Here we have that ‘μi​‘ is the mean of the values on the ‘i‘-th row, ‘γ‘ and ‘β‘ are learned constants and ‘ϵ‘ is a normalisation factor.

Different Formula for Standrad Deviation

In the above equation for ‘LayerNorm‘ we used the formula ‘σ=n1​⋅∑l=1n​(Ai,l​−μi​)2+ϵ​‘ for the standard deviation (plus the ‘ϵ‘ normalisation factor). We would like to avoid multiplying by ‘1/n‘ where possible because when dealing with quantised integers this introduces some error. To avoid this we use a different formula derived below, we write ‘μ^​i​:=n⋅μi​‘ and ‘νi​^​:=∑l=1n​Ai,l2​‘.

σ2⟹σ​=n1​⋅l=1∑n​(Ai,l​−μi​)2+ϵ=n1​⋅l=1∑n​(Ai,l2​−2⋅Ai,l​⋅μi​+μi2​)+ϵ=n1​l=1∑n​(Ai,l2​−2⋅Ai,l​⋅n∑k=1n​Ai,k​​+n2(∑k=1n​Ai,k​)2​)+ϵ=l=1∑n​nAi,l2​​−2⋅n∑k=1n​Ai,k​​⋅n∑l=1n​Ai,l​​+n2(∑k=1n​Ai,k​)2​+ϵ=n21​⋅(n⋅ν^i​−2⋅μ^​i2​+μ^​i2​+n2⋅ϵ)=nn⋅ν^i​−μ^​i2​+n2⋅ϵ​​.​

This allows us to pass ‘n⋅ν^i​−μ^​i2​‘ as input to the lookup table and it returns ‘n⋅ν^i​−μ^​i2​+n2⋅ϵ​1​‘.

Precision

When we calculate the term ‘n⋅ν^i​−μ^​i2​‘ to pass to the lookup we don't actually use the full value as the range of possible values it could take is far too large. Instead we only use the most significant ‘2bQ​‘ bits, where ‘bQ​‘ is the quantisation bit-length.

To see why we do this consider the following example. We pick two numbers ‘3.4372‘ and ‘4.5133‘ and their product is ‘15.51311476‘. If we round both to three significant figures then we have ‘3.44‘ and ‘4.51‘, their product is ‘15.5144‘ which we can see differs at the 5th significant figure, so using anything beyond this isn't going to give any more accuracy in the final result (as it is already wrong).

Quantised Evaluation

The main difficuly comes from computing the inverse square root term. For this we use a lookup table that takes as input ‘∑l=1n​n⋅Ai,l2​−μi2​‘ and outputs ‘Di​=(∑l=1n​n⋅Ai,l2​−μi2​+ϵ)−1/2‘. The final layer output is then calculated by performing the multiplication ‘γ⋅(n⋅Ai,j​−μi​)⋅Di​+β‘.

This output involves a degree 4 multiplication and so the bit size of the output is large, too large to apply our normal requantisation technique of applying ‘m=sin​/sout​‘ via its normal form ‘m=2−t⋅ϵ‘ using a fixed pointsclae factor. To still allow us to perform a requantisation we set ‘sout​‘ to be such that the requantisation requires only a right shift (i.e. we pick ‘sout​‘ such that ‘ϵ=1‘).

Proving the Layer

To prove the correct execution of LayerNorm we use a combination of lookups and standard sumchecks. The lookup protocol is used to prove correct computation of

Di​:=∑l=1n​n⋅Ai,l2​−μi2​+ϵ​1​​

and then a standard product sumcheck is used to prove that ‘γ⋅(n⋅Ai,j​−μi​)∗Di​+β=LayerNorm(A)i,j​‘ element-wise.

Throughout we use the fact that if ‘μ^​i​=∑l=1n​Ai,l​‘ then the polynomial that represents the tensor with each value on row ‘i‘ being ‘μ^​i​‘ (so if the original row is ‘Ai,1​,…,Ai,n​‘ this new tensor has row ‘μ^​i​,…,μ^​i​‘) can be computed by

μ^​(r1​,…,rk​)=b∈Bk​∑​2⌈logn⌉⋅eq(2−1,…,2−1,r⌈logn⌉​,…,rk​,b)⋅A(b)​

Step-by-Step

The prover recieves the input tensor ‘A‘ and its corresponding MLE ‘A(xˉ)‘. They use this to compute the input to the lookup table ‘LookupIn‘ and output of the lookup table ‘D‘ together with their corresponding MLEs ‘LookupIn(xˉ)‘ and ‘D(xˉ)‘.

To compute ‘LookupIn‘ the prover calculates ‘∑l=1n​n⋅Ai,l2​−μi2​‘. The outputs of this sum have large bit size, too large for a single lookup table, so if the quantisation bit length is ‘bq​‘ we use the fact that only the most significant ‘2bq​‘ bits of the sum have any real precision (any bits less significant than this are calculated via terms that involve rounding error). We split the most significant ‘2bq​‘ (see this for more info) bits of the sum off to become ‘LookupIn‘ and range check the remainder.

Now the prover commits to both ‘LookupIn(xˉ)‘, ‘D(xˉ)‘ and the range check lookup, appending the commitments to the transcript.

They run the lookup argument to obtain claims ‘LookupIn(s1′​ˉ​)=u‘, ‘D(s1′​ˉ​)=w‘ and ‘Range(s2′​ˉ​)=t‘. They then run another sumcheck to force all of these polynomials to be evaluated on the same point ‘s‘. We note that the point ‘s‘ has ‘⌈logn⌉‘ fewer variables than the polynomial ‘A(xˉ)‘ does.

Now that everything is evaluated on the same point they check via sumcheck that

2rangebits⋅LookupIn(sˉ)+Range(sˉ)=​b∈Bm​∑​eq(2−1,…,2−1,s,b)(n⋅2⌈logn⌉A(b)2−μ(b)2)​​

where each "row" of ‘μ(xˉ)‘ is the sum ‘∑l=1n​Ai,l​‘ repeated ‘2⌈logn⌉‘ times.

The prover also has the claim about the LayerNorm output which is a point ‘rˉ‘ and a value ‘v‘. They use this to reduce the claim on the output to a claim on ‘A(xˉ)‘ and ‘D(xˉ)‘ by running the sumcheck:

v=b∈Bm​∑​eq(rˉ,b)⋅(γ(b)⋅D^(b)⋅(n⋅A(b)−μ(b))+β(b))​​

Here ‘D^(xˉ)‘ is an extension of ‘D(yˉ​)‘ such that ‘D(yˉ​)=D^(a1​,⋯,a⌈logn⌉,rˉ​)‘ for any choice of the ‘ai​‘.

These two checks are batched together, the claim the prover creates on ‘D^‘ is verified by commitment opening and then we use the claims on ‘A‘ and ‘μ‘ to show that ‘μ(r)=∑b∈Bm​​2⌈logn⌉⋅eq(2−1,…,2−1,r⌈logn⌉​,…,rm​,b)⋅A(b)‘.

This final sumcheck provides use with the claim that is passed to the next layer.
