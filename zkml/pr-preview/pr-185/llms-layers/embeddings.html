Embeddings Layer

The embeddings layer implements a lookup table operation that maps discrete token identifiers to dense vector representations. This is a fundamental component in transformer-based language models.

Mathematical Formulation

Definition 1 (Embedding Map). An embedding map is a function E:T→Rd where:

  • T={0,1,…,v−1} is the finite set of token identifiers (vocabulary)
  • v∈N is the vocabulary size
  • d∈N is the embedding dimension

The embedding map can be represented as a matrix E∈Rv×d where the i-th row E[i,:] contains the embedding vector for token i.

Definition 2 (Embedding Layer Operation). Given an input sequence x=(x1​,x2​,…,xs​)∈Ts of length s, the embedding layer computes:

Embed(x)=(E[x1​,:],E[x2​,:],…,E[xs​,:])∈Rs×d

Inference Implementation

Input: Tensor x∈Zs×1 containing token identifiers

Parameters: Embedding matrix E∈Rv×d

Output: Tensor R∈Rs×d where R[i,:]=E[xi​,:]

The operation is equivalent to: R=H⋅E

where H∈{0,1}s×v is the one-hot encoding matrix defined by: H[i,j]={10​if j=xi​otherwise​

Zero-Knowledge Proof Protocol

Protocol Overview

The goal is to prove correct execution of the embedding lookup without revealing the input tokens. We use a matrix multiplication approach that avoids committing to the large output tensor if we were using a lookup protocol approach.

Commitment Phase

Commitment: The prover commits to the embedding matrix E using a polynomial commitment scheme, producing commitment CE​.

Proving Phase

Step 1: One-hot Encoding Construction The prover constructs the one-hot encoding matrix H∈{0,1}s×v as defined above.

Step 2: Matrix Multiplication Proof Using sumcheck protocol, the prover proves the relation: R=H⋅E

This produces two polynomial evaluation claims:

  1. Encoding claim: cH​:H~(r)=yH​ for random point r∈Flog(s)+log(v)
  2. Embedding claim: cE​:E~(r′)=yE​ for random point r′∈Flog(v)+log(d)

where H and E are the multilinear extensions of H and E, respectively.

Verification Phase

The verifier performs the following checks:

  1. Sumcheck Verification: Verify the sumcheck proof for the matrix multiplication.
  2. Embedding Claim: Verify cE​ using the polynomial commitment scheme opening.
  3. One-hot Claim: Directly evaluate H~(r) and verify cH​.

Efficient One-hot Evaluation

Theorem 1. The multilinear extension of the one-hot encoding matrix H can be evaluated in O(logs+logv) time.

Proof Sketch: The one-hot encoding has a highly structured form. For a single token (s=1), we have: H~(y)=β(x0​,y)

where β(a,y) is the multilinear extension of the indicator function for value a: β(a,y)=i=0∏logv−1​(ai​⋅yi​+(1−ai​)⋅(1−yi​))

where ai​ is the i-th bit of a in binary representation.

For the full sequence case, we can express the evaluation directly as: H~(r)=i=0∑s−1​β(i∥xi​,r)

where i∥xi​ denotes the concatenation of the binary representations of position i and token xi​, and r∈Flogs+logv is the evaluation point.

This can be computed in O(logs+logv) time using the Lagrange interpolation formula. □

Complexity Analysis

  • Prover Time: O(svd) for the sumcheck protocol
  • Verifier Time: O(logs+logv+logd)
  • Communication: O(log(svd)) field elements
  • Commitment Size: O(vd) for the embedding matrix
