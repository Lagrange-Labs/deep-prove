Multi-Head Attention

We define the multi-head attention (MHA) layer as the step in an LLM transformer that splits the Q, K and V matrices produced by QKV layer in multiple heads and combines them to produce a single matrix.

Description of the layer

Let's consider input matrices Q∈Rs×d, K∈Rs×d, V∈Rs×d, where s stands for the number of tokens being processed by the attention layer and d is an hyper-parameter of the model referred to as hidden dimension. Given a number of heads h defined by the model, each of these matrices is split in h sub-matrices over the column dimension: for instance, the matrix Q∈Rs×d is split in h matrices Qi​∈Rs×m, where m=hd​, such that the matrix Qi​ corresponds to the i-th chunk of m columns of matrix Q. Each of the sub-matrices Qi​, Ki​ and Vi​ belong to a head of the attention layer.

For each head, the following computation is performed.

Oi​=Softmax(m​Qi​∗KiT​​)∗Vi​

The output matrix Oi​∈Rs×m is the output of a single head. The output of the multi-head attention is then obtained by concatenating the output Oi​ of all the h heads in a single matrix O∈Rs×d

Masking

In LLM models like GPT-2, the attention layer is referred to as masked self-attention layer. We defer to this blogpost for the rationale of this design choice and its purpose. In terms of computation, the main difference in a masked self-attention layer is that, when computing the Qi​∗KiT​ product, the j-th row of Qi​ needs to be multiplied only with the first j−1 columns of matrix KT, rather than with all the columns.

In other words, the matrix QKi​=Qi​∗KiT​∈Rs×s is a lower triangular matrix, as only the entries QKi​[x,y] where x>=y corresponds to an actual product between a row of matrix Qi​ and a column of matrix KiT​. The entries QKi​[x,y] with x<y are instead filled with dummy values. Note that such dummy values should be big negative numbers rather than 0, as in this way these dummy values will not affect the output of the sub-sequent Softmax operation.

Proving

We now describe how to prove the linear operations in multi-head attention layer, deferring how to prove softmax to another page [ToDO: insert link when available].

The fundamental building block to prove linear operations in multi-head attention layer is a batched matrix multiplication protocol, which allows to simultaneously prove the matrix multiplications in all the h heads with a single sum-check. We start by describing how to simultaneously prove all the matrix multiplications QKi​=Qi​∗KiT​ for all the h heads.

The input of the operation are the matrices Q∈Rs×h∗m and K∈Rs×h∗m, which corresponds to the matrices produced by the QKV layer before being split in the h heads. The output matrix being produce is QK∈Rh∗s×s, that is we obtain a matrix where the i-th chunk of s rows corresponds to the output matrix Qi​∗KiT​ computed for the i-th head.

We consider input matrices to be padded to the next power on each of their dimensions. That is, the matrices have shape [s^,d], where s^=2⌈log2​(s)⌉ and d=2⌈log2​(h∗m)⌉. However, the proving protocol requires also h to be a power of 2, therefore we need to properly pad the columns of the input matrices Q and K.

Padding

Given h^=2⌈log2​(h)⌉ and m^=2⌈log2​(m)⌉, the number of columns of the input matrices must become d^=h^∗m^. This is done by padding the number of columns of each of the h heads to m^ and by adding h^−h dummy heads with s^ rows and m^ columns. This is achieved by multiplying each matrix Q∈Fs^×d, K∈Fs^×d, by the padding matrix P∈Fd×d^ defined as: P[i,j]={1if jmodm^<m∧⌊m^j​⌋<h∧i=⌊m^j​⌋m+jmodm^0otherwise​ In other words, the padding matrix P has h∗m non-zero entries, each yielding in the output matrix a corresponding non-zero entry in each row of the original matrices Q and K.

After multiplying the matrices Q and K with P, we get the padded matrices Q^​∈Fs^×h^m^ and K^∈Fs^×h^m^, respectively. Each of this matrix can now be split in h^ heads Q^​i​∈Fs^×m^, K^i​∈Fs^×m^; note that: Q^​i​∗K^iT​={Qi​∗KiT​ifi<h0otherwise​

Prove Proper Padding

We rely on sum-check to prove the construction of the padded matrices Q^​∈Fs^×d^ and K^∈Fs^×d^ from the matrices Q∈Fs^×d and K∈Fs^×d, respectively. For efficiency, we also prove at the same time the construction of the padded matrix V^∈Fs^×d^ from the input matrix V∈Fs^×d, which will be later needed in the multi-head attention layer.

The proving protocol starts from the following claims about the MLEs of output matrices Q^​, K^, V^:

  • Claim yQ^​​=Q^​(rqr​,rqc​), with rqr​∈Flog2​(s^), rqc​∈Flog2​(d^) being random points chosen by the verifier
  • Claim yK^​=K^(rkr​,rkc​), with rkr​∈Flog2​(s^), rkc​∈Flog2​(d^) being random points chosen by the verifier
  • Claim yV^​=V^(rvr​,rvc​), with rvr​∈Flog2​(s^), rvc​∈Flog2​(d^) being random points chosen by the verifier

Given these claims, the verifier samples random challenges λ1​, λ2​, which are shared with the prover. Then, given the padding matrix P∈Fd×d^, the proper padding of matrices Q, K and V is proven through sum-check protocol over the following relationship: yQ^​​+λ1​yK^​+λ2​yV^​=x∈{0,1}log2​(d)∑​Q(rqr​,x)P(x,rqc​)+λ1​K(rkr​,x)P(x,rkc​)+λ2​V(rvr​,x)P(x,rvc​) The sum-check protocol produces the following claims for a random point r∈Flog2​(d):

  • Claims P(r,rqc​), P(r,rkc​) and P(r,rkv​), which can be efficiently recomputed by the verifier
  • Claims yQ​=Q(rqr​,r), yK​=K(rkr​,r), yV​=V(rvr​,r) which are the claims about the input matrices of the multi-head attention layer

Prove Multi-Head Matrix Multiplication

Given the padded matrices Q^​∈Fs^×h^m^ and K^∈Fs^×h^m^, we simultaneously prove the product of all the h^ heads Q^​i​∈Fs^×m^, K^i​∈Fs^×m^ as follows.

Consider the output matrix QK^​∈Fh^s^×s^, given by the row-wise concatentation of the outputs of each of the h^ heads. In other words, considering this matrix as a 3d tensor (i.e., QK^​∈Fh^×s^×s^) then QK^​[i,j,k]=Q^​i​∗K^iT​[j,k].

The proving protocol starts from a claim y=QK^​(r1​,r2​,r3​), where:

  • r1​∈Flog(h^) is a random point chosen by the verifier
  • r2​∈Flog(s^) is a random point chosen by the verifier
  • r3​∈Flog(s^) is a random point chosen by the verifier

The prover now builds the vector b∈Fh^∗m^ such that b[i]=β(⌊m^i​⌋,r1​). The computation of QK^​ from matrices Q^​ and K^ is then proved with a sum-check protocol over the following relationship: \begin{equation} y = \sum_{x \in {0,1}^{\log_2(\hat{h}\hat{m})}} \mathbf{b}(x) \hat{Q}(r_2, x) \hat{K}(r_3, x) \tag{1} \end{equation} The sum-check protocol produces the following claims for a point r∈Flog2​(h^m^):

  • Claim b(r), which can be recomputed locally by the verifier
  • Claims yQ^​​=Q^​(r2​,r) and yK^​=K^(r3​,r), which can be used as the inputs claims in the protocol to prove the proper padding of matrices Q and K described above

Prove Masking

To apply masking on the output matrix QK∈Rh∗s×s, we need to make each of the h heads QKi​∈Rs×s a lower triangular matrix, where all the entries over the diagonal are replaced with a big negative value (referred to as −inf henceforth).

To prove this operation, we define a zeroifier matrix and a masking matrix:

  • The zeroifier matrix z∈Rs×s, where z[i,j]=0 if i<j, 1 otherwise
  • The masking matrix m∈Rs×s, where m[i,j]=−inf if i<j, 0 otherwise

Given matrices z and m, we can compute the masked QK matrix by performing the following computation on each of the h heads QKi​: QKi​⊗z+m, where ⊗ denotes the entry-wise matrix multiplication. In a nutshell, the rationale is that we first turn all entries to be masked to 0, and then we add −inf. The zeroing step is done to avoid potential overflows when summing the big negative number −inf.

When proving the masking operation, we need to consider that the masking is applied over the padded matrix QK^​∈Fh^×s^×s^. In this case, we need to mask all the s^−s dummy rows and dummy columns in each of the h^ heads QK^​i​∈Fs^×s^. Therefore, the padded zeroifier and masking matrices are computed as follows: z[i,j]={1if i<s∧j<s∧i>=j0otherwise​ m[i,j]={0if i<s∧j<s∧i>=j−infotherwise​ Since we need to apply the same matrices z and m over all the h^ rows of matrix QK^​∈Fh^×s^×s^, we can build corresponding matrices z^, m^∈Fh^×s^×s^, such that z^[i,j,k]=z[j,k] and m^[i,j,k]=m[j,k]. Note that the same relationship holds for the MLEs of the matrices.

The proving starts from a claim yM​=QK^​M​(r1​,r2​,r3​), where:

  • QK^​M​∈Fh^×s^×s^ is the output matrix, given by the row-wise concatenation of all the h^ heads after masking
  • r1​∈Flog(h^) is a random point
  • r2​∈Flog(s^) is a random point
  • r3​∈Flog(s^) is a random point

Given the claim yM​ and the MLE of the masking polynomial m, the prover computes the claim yZ​=yM​−m(r2​,r3​). Note that the claim m(r2​,r3​) should be efficienctly recomputable by the verifier as well.

The claim yZ​ refers now to the zeroified input matrix QK^​, so now we need to prove the zeroification of QK^​. Since this operation corresponds to an entry-wise multiplication of input matrix QK^​ with the extended zeroifier matrix z^∈Flog(h^×s^×s^), we can prove this computation via the following sum-check: yZ​=x∈{0,1}log(h^∗s^∗s^)∑​β(x,r)z^(x)QK^​(x) where r is the random point given by concatenation of r1​, r2​ and r3​. The sum-check protocol produces the following claims for a random point rs​∈Flog(h^∗s^∗s^):

  • Claim β(rs​,r), which can be efficienctly recomputed by the verifier
  • Claim z^(rs​), which can be efficienctly recomputed by the verifier: indeed, splitting rs​ in 3 points rs′​∈Flog(h^), rs′′​∈Flog(s^), rs′′′​∈Flog(s^), we have that z^(rs​)=z^(rs′​,rs′′​,rs′′′​)=z(rs′′​,rs′′′​), which can be efficienctly recomputed by the verifier
  • Claim QK^​(rs​). This is the claim about the unmasked matrix QK^​, which is the input of the protocol and can later be used as claim y in the sum-check protocol in Equation (1)

Prove Final Matrix Multiplication

After applying softmax over the masked matrix QK^​∈Fh^×s^×s^, we need to prove the final matrix multiplication of multi-head attention, which is multiplying each head of QK^​ with the corresponding head of the input matrix V^∈Fs^×h^×m^. More specifically, the output matrix O^∈Fs^×h^×m^ is given by the column-wise concatenation of its h^ heads O^i​∈Fs^×m^, where each Oi​^​=QK^​i​V^i​.

To perform this matrix multiplication, we need to consider the permuted matrix V∈Fh^×s^×m^ in place of V, as otherwise the matrices would have an incompatible shape when multiplied. We note that since V^[i,j,k]=V[j,i,k], then also for the MLEs of the matrices it holds that V^(x1​,x2​,x3​)=V(x2​,x1​,x3​); therefore, from a claim yV​=V(r), computed over a random point r=(r1​∈Flog(h^),r2​∈Flog(s^),r3​∈Flog(s^)), we can get a claim about V^ by simply swapping coordinates of the random point r, i.e., V^(r2​,r1​,r3​)=yV​.

To prove the multi-head matrix multiplication between QK^​∈Fh^×s^×s^ and V∈Fh^×s^×m^, we start from a claim yO^​=O^(rs​,rh​,rm​), where:

  • rs​∈Flog(s^) is a random point chosen by the verifier
  • rh​∈Flog(h^) is a random point chosen by the verifier
  • rm​∈Flog(m^) is a random point chosen by the verifier

Given this claim yO^​, the prover now computes the vector b∈Fh^×s^ such that b[i]=β(⌊s^i​⌋,rh​). The multi-head matrix multiplication is then proven with the following sum-check: \begin{equation} y_{\hat{O}} = \sum_{x \in {0,1}^{\log(\hat{h})}} \sum_{y \in {0,1}^{\log(\hat{s})}} \mathbf{b}(x, y) \hat{QK}(x, r_s, y)\widetilde{V}(x,y,r_m) \tag{2} \end{equation} The sum-check produces the following claims, for random points rx​∈Flog(h^), ry​∈Flog(s^):

  • Claim b(rx​,ry​), which can be efficienctly recomputed by the verifier
  • Claim QK^​(rx​,rs​,ry​), which can be employed as the claim yM​ in the masking proving protocol described above
  • Claim yV​=V(rx​,ry​,rm​), from which the verifier can get the claim yV^​ about the input matrix V^ as yV^​=V^(ry​,rx​,rm​)=yV​, because of the property about MLEs of V and V^ described earlier

Unpadding

The matrix O^ computed by the previous sum-check has shape [s^,h^∗m^], while the actual output matrix of the multi-head attention layer must have shape [s^,d], where d=2⌈log2​(h∗m)⌉. In general d≤h^m^, and so an unpadding operation might be necessary.

Recall that the matrix O^∈Fs^×h^×m^ is composed by the column-wise concatenation of h^ heads O^i​∈Fs^×m^. The unpadding operation needs to:

  • Get rid of the m^−m columns in each matrix O^i​
  • Discard the last h^−h padding heads

This operation can be performed by multiplying the matrix O^ with an unpadding matrix U∈Fh^∗m^×d computed as: U[i,j]={1if ⌊mj​⌋<h∧⌊m^i​⌋=⌊mj​⌋∧imodm^=jmodm0otherwise​ In other words, the unpadding matrix U has h∗m non-zero entries, each corresponding to a non-garbage entry in each row of the matrix O^, where we deem an entry as garbage if it is computed only from values adding by the initial padding of the multi-head attention layer.

The unpadding operation can be proven with a single sum-check. Given a claim yO​=O(rs​,rd​) for the output matrix O, where rs​∈Flog(s^) and rd​∈flog(d) are random points chosen by the verifier, the prover proves the following relationship by sum-check: yO​=x∈{0,1}log(h^m^)∑​O^(rs​,x)U(x,rd​) The sumcheck produces the followings claims for a random point r∈Flog(h^m^):

  • Claim U(r,rd​), which can be efficiently recomputed by the verifier
  • Claim O^(rs​,r), which can be employed as claim yO^​ in the sum-check protocol described in Equation (2)
